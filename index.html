TICKET #CR-014 · IDENTITY & GOVERNANCE
Hybrid Identity & Governance Lab Build
RESOLVED
EnvironmentM365 Developer Tenant (Lab)
SystemsEntra ID, Intune, Purview
PrioritySelf-Initiated
Duration2 Weeks

Summary

Designed and built a full hybrid identity and endpoint governance model from scratch in an isolated lab tenant, covering the complete path from account creation to compliance reporting — rather than studying each Microsoft service in isolation.

Actions Taken

  • Built the full user lifecycle in PowerShell against Microsoft Graph — creation, licensing, group assignment, quarterly access review enrollment, and full offboarding (session revocation, mailbox conversion, license reclaim).
  • Configured Conditional Access requiring MFA and device compliance, backed by Privileged Identity Management (PIM) for just-in-time admin roles instead of standing access.
  • Registered and tested both a SAML-based and an OAuth/OIDC-based enterprise application to compare authentication flows end-to-end, including claim/attribute mapping issues.
  • Enrolled a test device in Intune with compliance and configuration policies, then tied device compliance directly into Conditional Access.
  • Built a Purview DLP policy and retention label modeled on a real requirement from legal-sector client work — provable, unaltered record retention for sensitive documents.

Outcome

A fully lab-tested reference architecture connecting identity, access governance, endpoint compliance, and data protection — built to mirror how an audit or a security review would actually interrogate the environment, not just how each portal button works.

TICKET #CR-003 · AUTOMATION
PowerShell Onboarding & Offboarding Automation
RESOLVED
EnvironmentProduction — Corporate IT
SystemsActive Directory, M365
PriorityProcess Improvement
DurationOngoing

Summary

Repetitive, manual account lifecycle tasks were consuming time and introducing inconsistency between how different admins handled onboarding and offboarding.

Actions Taken

  • Built PowerShell scripts to standardize new-user provisioning — naming conventions, group membership, and license assignment — from a simple input list.
  • Automated offboarding steps (disable, session revocation, mailbox conversion, license removal) to remove reliance on memory or checklists during time-sensitive terminations.
  • Extended scripting into basic compliance auditing and reporting for IT operations reviews.

Outcome

Reduced manual variance in account lifecycle handling and cut the time-to-complete for both onboarding and offboarding — while making the process auditable rather than tribal knowledge.
